Providing Family Data


Details for providing family tree data

See: Definition of terms used for the family tree

The Priorities for data collection are:

  • "R" for required.
  • "C" for conditional depending on cases.
  • "P" for priority optional data.
  • "O" for optional data.

The data that can be collected in the database:

Data Title Description Priority R,C,P,O
Type Person's sex; male or female R
FirstName

A combination of the 'Given Names': FirstName + MiddleName.

Can include a NickName in quotes to create a unique identifier in cases where there are identical FirstName + MiddleName + LastName in the tree.

Some names in the tree can be place holders for people known to exist but the actual name is unknown. The name can contain place holder (TBD) names.

 R
LastName Can contain place holder (TBD) names. R
Suffix Capture suffix given at birth. Typically "Jr." or "II" or the like. Ignore "Sr", since that is implied but not the formal name. R
Nickname Nickname can be helpful in uniquely identifing the person. A Nickname will be very helpful in any occurance of identical full names ("FirstName + MiddleName + LastName + suffix"). The NickName can be added as "FirstName + MiddleName + "NickName" + LastName + suffix". P
Parent1 The Priority for Parent1 has cases:
  • Direct Descendant person: Required.
  • Spouse of a Direct Descendant: Optional.
  • Direct Descendants' recognized Stepchild: Required and must be the Direct Descendants' Spouse.
C
Parent2 The Priority for Parent2 has cases:
  • Direct Descendant person: Required.
  • Spouse of a Direct Descendant: Optional.
  • Direct Descendants' recognized Stepchild: Optional.
 C
Spouse A Spouse can be a marriage partner or a parenting relationship. R
BirthDate Any inforrmation known. Prefer Month-Day-Year but anything helps fill in the tree. P
DeathDate Prefer Month-Day-Year but anything helps fill in the tree. NOTE: A copy of Obituary can be included in the website. P
Photo Copy of the persons' photo. P
Title Professional prefixes are not particularly helpful for the tree but can be part of the person's narrative story. O
Birth place Helps with records search. O
Birth source Copy of Birth Certificate or other documentation. Helps with records search. Can be included in persons' narrative story. O
MarriageDate Helps with records search. Can be included in persons' narrative story. O
Marriage Place Helps with records search. Can be included in persons' narrative story. O
Marriage source Copy of ertificates or other documentation. Helps with records search. Can be included in persons' narrative story. O
Death place Helps with records search. O
Death source Copy of Death Certificate or other documentation. Helps with records search. Can be included in persons' narrative story. O
Burial date Prefer Month-Day-Year but anything helps fill in the tree. Typically addressed within an Obituary. Helps with records search. Can be included in persons' narrative story. O
Burial place Typically addressed within an Obituary. Helps with records search. Can be included in persons' narrative story. O
Burial source Copy of Death Certificate or other documentation. Helps with records search. Can be included in persons' narrative story. O
